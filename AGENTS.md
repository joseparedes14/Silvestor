# AGENTS.md - Instrucciones para el agente

## Logging de cambios

Siempre que realices cambios en el códigobase (nuevas funcionalidades, refactors, bugfixes, cambios en la base de datos, etc.), debes registrar los cambios en el archivo `docs/changelog.md` siguiendo el formato existente:

- Añade una nueva entrada con la fecha y versión correspondiente.
- Clasifica los cambios bajo las secciones adecuadas: `### Añadido`, `### Cambiado`, `### Corregido`, `### Eliminado`.
- Describe cada cambio de forma clara y concisa para que un usuario del proyecto pueda entender qué se modificó y por qué.
